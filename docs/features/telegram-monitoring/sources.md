# Telegram Monitoring — список источников (prod/test)

Канонический список Telegram‑источников для мониторинга хранится в машинно‑читаемом виде:

- `docs/features/telegram-monitoring/sources.yml`

Этот список используется для:

- сидов/миграций SQLite (автодобавление отсутствующих источников при старте бота);
- кнопки UI `/tg` → «🧩 Синхронизировать источники» (ручная синхронизация);
- задания уровней доверия и дефолтов (`trust_level`, `default_location`, `festival_series`) без дублей.

## Поля

- `trust_level` (low/medium/high) — влияет на Smart Update (приоритет/конфликты).
- `default_location` — подставляется, если пост не содержит адрес/площадку.
- `festival_series` — помечает канал как фестивальный (`telegram_source.festival_source=1`).
- `filters` — server-side фильтры (например, отсечка регулярных экскурсий или «#реклама»).
  - `bridge_notice_daily` — для официального `@klgdcity`: развод/разводка мостов должен импортироваться как событие и дополнительно уходить notice в `/daily` каналы после успешного `event_id`.

## Быстрые проверки

- В UI: `/tg` → «📋 Список источников» (есть пагинация при большом количестве).
- Чтобы привести тест/прод к канонике: `/tg` → «🧩 Синхронизировать источники».

Примечание: если удалить канонический источник через UI, он будет добавлен обратно при следующей синхронизации (или при перезапуске бота). Для временного отключения используйте `Disable`.
